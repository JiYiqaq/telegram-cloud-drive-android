# 网盘式主界面与可靠删除设计

## 目标

本次改版把现有“文件与传输记录堆在同一长页面”的主界面调整为更接近主流网盘的三入口结构：文件、传输、设置。文件入口承担浏览、搜索、排序、上传、新建文件夹与批量管理；传输入口集中展示进行中和历史记录；设置仍使用现有独立页面。视觉上采用干净的蓝绿色品牌头部、卡片式快捷操作、清晰的文件类型图标和固定底部导航，不复制百度网盘的名称、图标或品牌素材。

## 信息架构与交互

“文件”页顶部展示当前空间标题和设置入口，下面是圆角搜索框、路径面包屑以及上传/新建文件夹快捷卡片。文件列表保留现有安全能力：点击文件下载、点击文件夹进入、更多菜单中提供详情、重命名、移动和删除，长按或“管理”进入批量模式。右下角主操作按钮展开上传文件和新建文件夹，空目录也提供明显上传入口。

“传输”页按进行中与历史记录分区。进行中任务显示进度、速度及取消操作；失败和取消任务显示重试与清理；上传或下载成功后显示完成状态并允许删除该条记录。历史区提供“一键清理已完成/失败/取消记录”。删除传输记录只删除本地队列历史，不删除云端文件；活动任务不能直接移除，必须先取消，以免后台 Worker 丢失状态。

## 删除失败修复

现有上传完成逻辑允许索引状态为 `DIRTY`，但文件删除入口只接受 `SYNCED`。因此用户可能看到“上传成功”，随后删除立即失败。删除启动器将识别 `INDEX_CONFIRMATION_REQUIRED`：先运行现有原子索引更新，再重试一次删除入队；其他错误保持原样，不做盲目重试。远端分块删除仍遵循先确认 Telegram 消息删除、再发布新索引、最后清理本地元数据的安全顺序。若 Bot 缺少删除权限或网络结果不确定，文件恢复为“删除未完成”并保留重试入口。

## 测试与发布

新增纯 Kotlin 回归测试覆盖索引未同步时的“同步一次后重试”、非索引错误不重试、第二次失败透传；新增传输记录清理策略测试，验证只有 `SUCCESS`、`FAILED`、`CANCELED` 可移除。Room 仓储增加终态单删与批量清理接口，AndroidTest 编译覆盖数据库签名。UI 通过 Compose 编译、Lint、完整单元测试和 Release APK 构建验证。版本升级后按既定流程推送主分支、等待 GitHub Actions，并发布带 SHA-256 的签名 APK。
