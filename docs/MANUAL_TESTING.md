# 人工测试清单

以下流程需要真实 Android 设备、用户自己的 Telegram Bot 和私人频道。不要在截图、Issue、日志或测试记录中保存真实 Token、Chat ID、同步密码或频道内容。

## 测试准备

- 使用与个人数据隔离的测试 Bot 和私人测试频道。
- 将 Bot 设为频道管理员，并授予发消息、删消息和置顶消息权限。
- 准备小文件、超过一个分块的大文件、零字节文件、同名文件和多个 MIME 类型文件。
- 准备第二台设备或清除应用数据，用于恢复验证。
- 测试前记录频道中原有置顶消息，避免覆盖无关置顶内容。

## 首次配置

- 无效 Token 显示简体中文错误，且界面、日志、通知不回显完整 Token。
- 有效 Token 显示 Bot 名称和用户名。
- 无效 Chat ID、非频道 Chat ID、权限不足分别给出可理解提示。
- 频道测试发送并删除临时消息，不残留测试数据。
- 自动检测只展示 `channel_post` 候选，须由用户确认后填入。
- 两次密码不一致、密码过短时不能初始化；密码输入页面禁止系统最近任务截图。
- 初始化成功后频道中存在合法的加密置顶索引。

## 上传与目录

- 通过系统文件选择器一次选择一个和多个任意类型文件。
- 小文件、大文件和零字节文件均可上传；频道消息不暴露原文件名、目录名或用户信息。
- 大文件按设置中的分块大小顺序上传，进度、速度、当前分块和总分块数持续更新。
- 断网后任务进入等待网络状态，联网后继续；限流时按 Telegram 返回的等待时间重试。
- 取消上传后文件不进入正式索引；选择清理时已上传分块被删除或标记为待清理孤立块。
- 新建多级目录、重命名、移动和返回上级均保持层级正确；禁止把目录移动到自身或子目录。
- 搜索和名称/大小/时间正反排序结果正确。

## 下载与完整性

- 下载通过系统“创建文档”选择目标位置，不要求全盘存储权限。
- 多分块文件按顺序下载并解密，完成后 SHA-256 与详情页记录一致。
- 断网、取消、重试不会把不完整文件标记成功。
- 使用错误同步密码恢复、篡改密文块或截断响应时停止输出并显示认证/完整性错误。
- 目标 Uri 失效或空间不足时给出失败状态，不覆盖已有云端元数据。

## 删除与恢复

- 删除文件前出现明确确认；所有块和新索引提交成功后，文件才从目录消失。
- 中途断网或部分 Telegram 消息删除失败时显示“部分删除”，重试只处理剩余块。
- 删除非空文件夹需要二次确认，并递归处理其中内容。
- 置顶消息不是 TeleDrive 索引时，应用拒绝覆盖并提示用户处理。
- 第二台设备使用相同 Token、Chat ID 和同步密码可恢复完整目录、文件和分块映射。
- 错误密码、损坏索引或不支持的索引版本不会覆盖当前本地缓存。

## 设置、退出与系统行为

- 跟随系统、浅色、深色三种主题立即生效并在重启后保留。
- 4/8/12/18 MiB 分块设置仅影响新上传任务。
- 更新连接配置时，只有 Bot 和频道验证全部成功才替换原配置。
- 导出诊断不包含 Token、Chat ID、同步密码、密钥、文件名或 Telegram 下载 URL。
- “退出并清除凭据”取消任务、清除 Room 缓存和临时文件、删除 Keystore 条目并返回首次配置页。
- Android 8.0（API 28）与当前 Android 稳定版至少各完成一轮冒烟测试。
- 后台上传/下载通知可见，取消按钮有效；进程被系统终止后 WorkManager 状态可恢复。
